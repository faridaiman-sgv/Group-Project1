## 1. Introduction

Docker is a platform that enables developers to package applications and their dependencies into containers. Containers provide a lightweight and efficient environment that allows applications to run consistently across different systems.

In this project, Docker technology is used to deploy multiple containers consisting of a MySQL database, Python socket servers, Python socket clients, C socket servers, and C socket clients. All containers are connected through the same Docker network, allowing them to communicate efficiently using container service names instead of IP addresses.

The project demonstrates the concepts of:

* Socket Programming (TCP, in both Python and C)
* Docker Containerization
* Docker Compose multi-container orchestration
* Docker Networking
* Database Integration with MySQL
* Multithreading / multi-client handling

Python and C programming languages are used to implement both TCP socket servers and TCP socket clients, while MySQL is used to store and manage the points data generated by the servers.

The system is designed so each server automatically increments its own user's points every 30 seconds, and each paired client polls its server every 10 seconds to retrieve and display the latest points and timestamp.

## 4. System Architecture

The system consists of **13 containers**:

### Database Container
* MySQL Database (`mysql-db`)

### Python Server Containers
* python-server1 (port 5001)
* python-server2 (port 5003)
* python-server3 (port 5005)

### C Server Containers
* c-server1 (port 5002)
* c-server2 (port 5004)
* c-server3 (port 5006)

### Python Client Containers
* python-client1 → python-server1
* python-client2 → python-server2
* python-client3 → python-server3

### C Client Containers
* c-client1 → c-server1
* c-client2 → c-server2
* c-client3 → c-server3

All containers are connected through the Docker network:

```text
itt440-network
```

### Communication Flow

1. All servers connect to the MySQL database (`mysql-db`), waiting for its healthcheck to pass before starting.
2. Each server updates its own user's row every 30 seconds in a background thread.
3. MySQL stores each user's points and timestamp in the `user_points` table.
4. Each client connects to its paired server over TCP, sends the command `GET_POINTS`, and receives a JSON response.
5. The Docker network allows every container to reach every other container by service name (e.g. `mysql-db`, `python-server1`, `c-server2`).

## 6. Docker Compose Configuration

Docker Compose is used to:

* Build the Python and C server images
* Create the shared Docker network (`itt440-network`)
* Map each service's ports
* Start services in the correct order using `depends_on` and a MySQL `healthcheck`
* Inject per-container environment variables (port, username, DB credentials)

### docker-compose.yml (summary)

```yaml
services:
  mysql-db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: itt440_db
      MYSQL_USER: itt440_user
      MYSQL_PASSWORD: itt440_pass
    ports: ["3306:3306"]
    volumes:
      - mysql-data:/var/lib/mysql
      - ./mysql-init:/docker-entrypoint-initdb.d
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "itt440_user", "-pitt440_pass"]
      timeout: 20s
      retries: 10

  python-server1:
    build: { context: ., dockerfile: Dockerfile.python-server }
    ports: ["5001:5001"]
    depends_on:
      mysql-db: { condition: service_healthy }
    environment:
      SERVER_PORT: 5001
      USER_NAME: python_user

  c-server1:
    image: gcc:latest
    command: sh -c "apt-get update && apt-get install -y -qq default-libmysqlclient-dev libjson-c-dev && gcc -o c_server c_server_simple.c -lmysqlclient -ljson-c -pthread && ./c_server"
    ports: ["5002:5002"]
    depends_on:
      mysql-db: { condition: service_healthy }
    environment:
      SERVER_PORT: 5002
      USER_NAME: c_user

  python-client1:
    image: python:3.9-slim
    command: sh -c "python python_client.py"
    depends_on: [python-server1]
    environment:
      SERVER_HOST: python-server1
      SERVER_PORT: 5001

  c-client1:
    image: gcc:latest
    command: sh -c "gcc -o c_client c_client_final.c && ./c_client"
    depends_on: [c-server1]
    environment:
      SERVER_HOST: c-server1
      SERVER_PORT: 5002

networks:
  itt440-network:
    driver: bridge

volumes:
  mysql-data:
```

*(python-server2/3, c-server2/3, and their matching clients follow the same pattern with different ports and usernames — see the full `docker-compose.yml` in the repository for all 13 services.)*

### 7.3 C Socket Servers

| Server | Port | Identity |
|---|---|---|

|---|---|
| python-client1/2/3 | python-server1/2/3 |

* Connect over TCP and send the `GET_POINTS` command
* Parse and print the JSON response (user, points, timestamp)
* Retry automatically every 10 seconds, tolerating the server not being ready yet

## 8. Dockerfiles

### Python Server Dockerfile (`Dockerfile.python-server`)

```dockerfile
FROM python:3.9-slim

WORKDIR /app

RUN pip install --no-cache-dir mysql-connector-python

COPY python_server.py .

EXPOSE 5001

CMD ["python", "python_server.py"]
```

### C Server Dockerfile (`Dockerfile.c-server`)

```dockerfile
FROM gcc:latest

RUN apt-get update && \
    apt-get install -y \
    default-libmysqlclient-dev \
    libjson-c-dev \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

COPY c_server_simple.c .

RUN gcc -o c_server c_server_simple.c -lmysqlclient -ljson-c -pthread

EXPOSE 5002

CMD ["./c_server"]
```

*Note: in `docker-compose.yml`, the three C servers and three C clients are actually built and compiled inline via their `command:` field rather than through `Dockerfile.c-server`; the Dockerfile is kept in the repo as a standalone, reusable build path.*

## 11. Problems Faced

| Problem | Solution |
|---|---|
| C servers/clients need libraries not present in the base `gcc:latest` image | Installed `default-libmysqlclient-dev` and `libjson-c-dev` at container start (or via `Dockerfile.c-server`) before compiling |
| Servers starting before MySQL was ready | Added a `healthcheck` to `mysql-db` and `depends_on: condition: service_healthy` on every server |
| Containers unable to resolve each other by name | Connected every container to the same custom bridge network (`itt440-network`) instead of relying on default networking |
| C client using `localhost` instead of the container hostname | Used the Docker Compose service name (e.g. `c-server1`) via the `SERVER_HOST` environment variable, resolved with `gethostbyname()` |
| Multiple servers needing distinct identities from one codebase | Passed `SERVER_PORT` and `USER_NAME` as environment variables per service, rather than hardcoding them |
| Dropped MySQL connections in long-running C threads | Called `mysql_ping()` before each query and reconnected automatically if the ping failed |
